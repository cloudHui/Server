# 项目开发规范

后续全项目代码统一遵守以下约定：

1. 每个新增或修改的方法控制在 50 行以内；复杂逻辑拆成职责单一的私有方法。
2. 每个新增的业务方法、关键分支和协议转换必须有中文注释，说明原因和数据流向；简单 getter/setter 不强制添加注释。
3. 优先复用现有公共脚本、样式、服务、线程工具和协议，不复制粘贴同一套逻辑。
4. 网络线程只负责接收和发送；游戏状态、桌子管理、玩家状态和数据库写入不得直接阻塞网络线程。
5. 每桌的牌局、定时器、玩家增删和玩家操作必须进入同一个桌子专属串行执行器；不同桌子可以并行。
6. 桌子生命周期使用独立的桌子管理执行器；数据库使用独立数据库线程池，不与桌子线程或网络线程混用。
7. 跨线程需要等待或返回结果时使用 `CompletableFuture` 回调，禁止通过 `sleep`、忙等或共享可变对象同步。
8. 不保留无效循环、重复判断、未使用变量和临时代码；提交前执行 `git diff --check`。
9. 前端路径统一使用 `appUrl()`，保证根路径和随机 context-path 都能正常访问。
10. 修改后至少执行对应的 Java 构建和前端脚本语法检查，并在提交说明中写清楚功能、修复和验证结果。
11. Git 提交作者使用 `liuyunhui`，提交信息要具体描述本次改动，不使用无意义的“更新代码”。
