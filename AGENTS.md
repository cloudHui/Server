# 项目协作规范

- Git 提交描述必须使用中文，清晰说明本次变更内容。
- 未经明确要求，不修改现有 WebSocket 通信链路或新增通信协议。

## 代码复用与拆分

- 优先复用已有公共模块、工具、样式、脚本、服务与协议；禁止复制粘贴出第二份同类逻辑。
- 新增能力前先检索 `shared/`、同玩法包、已有 Handler/Service；能扩展则扩展，不平行再写一套。
- Java：单个类职责膨胀或文件过长时，按领域拆分包/拆分类（如格式化、结算、出牌、叫分）；新增/修改方法尽量控制在约 50 行内，复杂分支拆成私有方法。
- 前端：单个 HTML 页面过长时，样式抽到独立 css、逻辑抽到独立 js，跨页公共能力放到 `web/.../static/shared/`；玩法专属逻辑按玩法文件拆分，不把多玩法堆在同一巨型页面。
- 拆分后保持调用链清晰、避免循环依赖；行为与协议兼容，不借拆分之机改通信契约。
