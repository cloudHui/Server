# Web 开发规范

后续修改 `web` 模块统一遵守以下约定：

1. 每个新增或修改的方法控制在 50 行以内；复杂逻辑拆成职责单一的私有方法。
2. 每个新增的业务方法、关键分支和协议转换必须有中文注释，说明原因和数据流向；简单 getter/setter 不强制添加注释。
3. 优先复用现有页面公共脚本、样式、服务和协议，不复制粘贴同一套逻辑。
4. 页面只在真正需要时请求数据；入口页面不提前请求游戏房间、牌桌或小游戏状态。
5. 不保留无效循环、重复判断、未使用变量和临时代码；提交前执行 `git diff --check`。
6. 前端路径统一使用 `appUrl()`，保证根路径和随机 context-path 都能正常访问。
7. 修改后至少执行对应的 Java 构建和前端脚本语法检查，并在提交说明中写清楚功能、修复和验证结果。
8. Git 提交作者使用 `liuyunhui`，提交信息要具体描述本次改动，不使用无意义的“更新代码”。
